# 农业 Agent 系统：可行性分析与 MVP 设计

> 日期：2026-06-24

---

## 一、项目可行性总评

### 结论：**可行，但需要极其克制的切入方式**

| 层级 | 组件 | 个人/小团队可行性 | 所需时间 |
|------|------|-------------------|----------|
| L1 | MCP 协议桥接层 | **完全可行** | 1-2 周 |
| L2 | Agent Router + 调度 | **可行** | 1-2 周 |
| L3 | 轻量数字孪生模拟器 | **可行（简化版）** | 2-4 周 |
| L4 | 垂直领域模型微调 | 需要数据 + GPU 资源 | 1-3 月 |
| L5 | 完整数字孪生（含作物模型） | 需要领域专家合作 | 3-6 月 |
| L6 | 跨模态数据总线 | 需要工程团队 | 6+ 月 |

---

## 二、核心问题：MCP 桥接 vs 数字孪生模拟器

### 建议：**先 MCP 桥接，数字孪生作为 MCP Server 的一种特殊实现**

```
方案 A（先模拟器）：  模拟器 → ??? → Agent → 决策
方案 B（先 MCP）：    真实数据源 ─┐
                                  ├─ MCP协议 → Agent Router → 决策
                       模拟数据源 ─┘
```

方案 B 天然兼容两种数据源。MCP 桥接建好后，数字孪生模拟器只是"另一个 MCP Server"。

---

## 三、MVP 架构

```
用户查询 → Agent Router → 传感器MCP / 气象MCP / 知识库MCP → 数据汇聚 → 诊断报告
```

## 四、30 天行动计划

| 周 | 目标 | 产出 |
|----|------|------|
| W1 | 搭建 MCP 基础设施 | 3 个可运行的 MCP Server |
| W2 | 实现 Agent Router | 意图识别 + 工具链调度 |
| W3 | 端到端诊断流程 | 多源数据汇聚 → 结构化报告 |
| W4 | 传感器模拟器 | 简化版数字孪生种子 |

## 五、风险

- **高风险**：领域知识缺口、数据质量
- **中风险**：模型幻觉、API 稳定性
- **低风险（MVP 可忽略）**：并发、部署、多租户
