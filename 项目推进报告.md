## 四、功能树

```
用户输入
  │
  ▼
Orchestrator（编排层）
  ├─ 上下文抽取（作物、意图、温室）
  │
  ├─ 感知层
  │   ├─ 视觉 Agent ─── Swin-Tiny / HuggingFace 模型
  │   ├─ 传感器 Agent ── 三层异常检测（IsolationForest + LSTM + Chronos）
  │   └─ 气象 Agent ─── OpenWeatherMap / 模拟
  │
  ├─ 专家层
  │   ├─ 病理 Agent
  │   ├─ 栽培 Agent
  │   └─ (预留更多)
  │
  ├─ 记忆层
  │   ├─ RAG ─────── Qdrant → 内存知识库（自动降级）
  │   ├─ KG ──────── Neo4j + AgriKG → DISEASE_DB（自动降级）
  │   └─ 案例库 ───── 模式匹配 + SQLite 决策回灌
  │
  ├─ Debate & Judge
  │   ├─ 规则版 Judge
  │   ├─ DeepSeek Judge（结构化裁决）
  │   ├─ Critic 反驳
  │   └─ 置信度校准 + 硬约束校验
  │
  └─ 输出层
      ├─ 结构化诊断报告
      ├─ Gateway API
      ├─ Streamlit UI
      └─ TUI 终端界面
```

- 效果：灌溉+病害场景下 Critic **稳定触发**（不再依赖强制高湿）；规则模式降权气象Agent 0.72→0.43，LLM 模式 0.72→0.22，Judge 引用 KG 三元组裁决。
