# MVP 架构升级方案 — 模型选型更新

> 基于 Hugging Face + GitHub 调研 | 2026-06-27

## 一、视觉模块：ResNet-50 → ViT 或多模态 VLM

### 原方案 vs 新方案

| | 原方案：ResNet-50 | 新方案 A：ViT-Base | 新方案 B：Swin-Tiny | 新方案 C：Qwen3-VL-2B |
|---|---|---|---|---|
| 准确率（PlantVillage） | 96-98% | 98-99.5% | 98-99.5% | — |
| 真实田间准确率 | 60-70% | 72-80% | 75-82% | 不适用（端到端） |
| 输出格式 | 分类标签 | 分类标签 | 分类标签 | **自然语言诊断** |
| 模型大小 | 25.6M | 86M | 28M | ~2B（GGUF ~1.2GB） |
| CPU 推理 | 15-40ms | 30-80ms | 40-100ms | 2-5 tok/s |
| Hugging Face 可用模型 | 有 | **有（直接可用）** | **有（直接可用）** | **有（GGUF 量化）** |

### 推荐：Swin-Tiny 作为主模型

理由：
- 真实田间准确率比 ResNet 高 10-15 个百分点（75-82% vs 60-70%）
- 层级化 Transformer 架构，天然适合捕捉叶片病斑的空间层次特征
- 模型只有 28M 参数，比 ViT-Base（86M）更轻量
- Hugging Face 上有现成微调模型：`gianlab/swin-tiny-patch4-window7-224-finetuned-plantdisease`

**备选（前沿方向）：Qwen3-VL-2B 多模态 VLM**

这是一个更激进但可能更实用的方案——用多模态视觉语言模型替代传统分类器：

```
传统分类器：图片 → 病害类别（如"番茄叶霉病"）→ 然后查知识库拿防治方案
多模态 VLM：图片 → 直接输出"叶片正面出现淡黄色病斑，背面有灰色霉层，
           疑似番茄叶霉病。建议立即加强通风，喷施多菌灵500倍液..."
```

Hugging Face 上已有量化版本：
- `mradermacher/Plant-Disease-Qwen3VL-2B-GGUF`（CPU 可跑）
- `nasko71/LLaVA-v1.5-7B-Plant-Leaf-Diseases-Detection-Q4_K_M-GGUF`（更强但更大）

这意味着 **ResNet + 知识库检索** 两步可以合并成一步，由 VLM 直接输出完整诊断。

### 具体模型选择

```python
# 方案 A：传统分类器（轻量、快速）
from transformers import pipeline
classifier = pipeline("image-classification",
    model="gianlab/swin-tiny-patch4-window7-224-finetuned-plantdisease")

# 方案 B：番茄专用 ViT
classifier = pipeline("image-classification",
    model="wellCh4n/tomato-leaf-disease-classification-vit")

# 方案 C：多模态 VLM（需 llama-cpp-python）
# pip install llama-cpp-python
# 下载 GGUF 模型后直接推理
```

## 二、传感器异常检测模块：LSTM → LSTM-Autoencoder + Chronos 双通道

### 原方案 vs 新方案

| | 原方案：LSTM | 新方案 A：LSTM-Autoencoder | 新方案 B：Chronos-T5-Small | 新方案 C：Isolation Forest |
|---|---|---|---|---|
| 训练数据需求 | 需要标注的异常数据 | **只需正常数据** | **零标注（零样本）** | 少量即可 |
| 检测原理 | 分类 | 重构误差 | 预测偏差 | 密度离群 |
| 多变量支持 | 是 | 是 | 逐变量 | 是 |
| CPU 可用 | 是 | 是 | 是（~185MB） | 是（极快） |
| 实现复杂度 | 中 | 中 | **极低（pip install）** | **极低** |
| 适合场景 | 有标注数据 | 有正常运行数据 | **没有任何标注数据** | 快速基线 |

### 推荐：三层防御架构

不需要选一个，三个互补使用：

```
传感器数据流入
      │
      ├─→ [第1层] Isolation Forest ← scikit-learn
      │   极快，先过滤明显异常（传感器故障、极端值）
      │   延迟：<1ms
      │
      ├─→ [第2层] LSTM-Autoencoder ← PyTorch
      │   学习正常模式，发现偏离正常模式的异常
      │   捕捉多变量交叉关系（温度升+湿度升=不合物理规律）
      │   延迟：<5ms
      │
      └─→ [第3层] Chronos-T5-Small ← Amazon 时序基础模型
          零样本预测，预测未来值，偏差大则告警
          作为"第二意见"验证前两层的结果
          延迟：~50ms CPU

      ↓ 汇聚三层结果
      异常分数 + 异常类型 + 置信度
```

### 最快可跑通的路径

今天就可以做的两件事（不需要 GPU，不需要训练数据）：

**视觉模块** — 3 行代码接入：
```python
from transformers import pipeline
classifier = pipeline("image-classification",
    model="gianlab/swin-tiny-patch4-window7-224-finetuned-plantdisease")
result = classifier("tomato_leaf.jpg")  # 直接用，不需要训练
```

**异常检测模块** — 零样本：
```python
pip install chronos-forecasting
# 然后直接用 Chronos-T5-Small 做预测-对比，不需要任何训练
```

这两个加起来，今天的 MVP 就能从"模拟演示"升级成"真实模型驱动"。
